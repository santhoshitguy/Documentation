Deployment

Angular for Production

After completing the development, we need to deploy the application to the end user test. For that Angular cli have some inbuilt commands to generate the dist for deployment. The Ahead-of-Time(AOT) which helps to load pre-compiled templates on the page loading time. It'll catch all errors while compile, so we can increase the performance.

The AOT compilation, downloads pre-compiled version of the application. So the user can reach out to any page or perform any logic immediately without any delay.

ng build --prod

We need to copy the files into a server sub-domain, and need to attch the build flag, --base-href and add attribute appropriately. For example, if the index.html is on the server at /demo/demo1/index.html, set the base href to like this.

Another way of setting the --base-href is using the build command

ng build --prod --base-href="/my/app/"

For more detailed information, visit this official Angular documentation website.

Ahead-of-time (AOT) vs just-in-time (JIT)

Actually Angular has only one compiler. The major difference between these two approaches are timing and the set of tools used for it. Tha AOT is compiles once at the build time using set of libraries, and the other side the JIT is compiles every user reach out the particular page using another set of libraries. To enable the JIT compiler,

ng build --prod --aot=false
