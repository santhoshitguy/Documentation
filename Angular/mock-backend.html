Mock Back-end

Overview

Anglar theme uses the mock backend which uses in memory web api to replicate the real server API like response to make easy the real server connectivity while developing the application.

To use this technology, we need Http and HttpInterceptor, HTTP_INTERCEPTORS to interpret particular API calls and generate the response from Frontend itself.

How switching to the Real REST API

This fake backend you can locate in the following path ./src/app/core/helpers/fake-backend.ts

When starts to gain control of the real APIs then we have to create a new service which can communicates with the real server and need to transfer all our calls to that service.

Before make ready the built for deploy in server, we need to change api endpoint in the environments file.

For example, consider the following services:

Hide or delete the file path which points to the fake backend server.

// export { AuthHTTPService } from './auth-fake-http.service';

And then import the newly created service which points to the real server endpoint.

export { AuthHTTPService } from './auth-http.service';
