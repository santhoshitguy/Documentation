Angular Documentation

Overview

Angular is a most popular and fast growing single page application front-end JavaScript framework. This Anglar theme is build on top fo the latest Angular Front-end Javascript framework and the latest Bootstrap. This theme not only a Admin Template to start your Saas Application or enterprise application and also good platform to start learn the angular from basics to an expert level by using the comments on each component and modules.

Angular File Structure

Anglar theme is well structured in both design as well in code. It is completely designed and developed at the developers in preference. It has all things need to develop and kick start the application in a very less time with efficient code and reusability.

Folder Name Description
dist The dist folder contains the deployable files in the server which means compiled files of the entire application. To generate this Run the command ng build --prod or npm run build --prod
e2e It contains the end to end testing related files which are generate using Protector. Visit link https://angular.io/cli/e2e for more info.
node_modules The node_modules contains all the packages used to run the application or the theme. package.json is the file which maintain all the packages used. Run the following command npm install
src The entire application lives in the src folder only. All custom angular components, directives, pipes and modules are generated and communicated in this location only. Anyother files are outside consider to be supporting the application or theme.
- app Main application folder. Root for all other components or modules
-- app.component.html The root component of the theme or application.
-- app.component.ts Class of the root component.
-- app.component.scss Styles of the root component.
-- app.module.ts Root module of the application or theme. The place where we need to import all required modules and components.
-- core Contains all core functionalities of the application
--- directives Contains all customized directives to use overall theme or application
--- guards Guards to protect the users or malicious threats to access the specific component or modules.
--- helpers Contains the fake backend, jwt-interceptor and error interceptors which are supports to mock the fake database access.
--- menu Menu service which helps to render the left side menu list dynamically.
--- preloader Preloader service used for make ready the Preloader component at the time of logging in or loading other pages.
--- services Lists all of the customized services for different purposes.
--- theme-settings Theme settings like layout and etc.
--- themes Color theme settings like change the colors of the theme or application.
--- translator Translator service helps to handle the language settings over the application.
-- layout Layout folder is contains all different parts of the application or theme.
--- header It shows the main header of the application and icons like settings and notifications etc.
--- secondary-header Contains the page title as well some additional description about that component
--- sidebar Main sidebar component of the application or theme which contains the all menu list.
--- offsidebar Offsidebar contains the theme options, layout settings and some other additional information about the theme or application.
--- footer Global footer component
-- routes This module contains all custom components which are separate routes of the application.
--- apps Some of the pre-built applications like mailbox, and todo list.
--- blog Blog module has all the components related to a blogging application or website.
--- charts Chart module lists all different kind charts for various purposes.
--- dashboard Contains all available dashboard pages for various categories.
--- ecommerce E-Commerce module lists all components needed for an e-commerce application
--- elements Elements module carries all global elements like buttons, badges, cards, grids, pagination and etc.
--- events Event module has all the components to make events conducting application.
--- forms In angular we can deal with all kind of form related functionalities. Forms module contains advanced, basics, template-driven, reactive form components.
--- gallery Gallery for showcase the images albums.
--- home Home module has the main home screen component
--- maps Maps module helps to list different kind of maps like google maps.
--- metrics List some metrics related components
--- miscellaneous Miscellaneous module holds a lot of components which are not mandatory but required in most of the applications like, Clipboard, code-editor, idle-timer, pdf-viewer and more.
--- ngrx Module based on the Ngrx store concept.
--- pages Some common pages like 404, 500, login, lock and etc.
--- pipes Lists some custom pipes
--- tables Different types of tables
--- views List of views for various purposes.
--- widgets List of widgets used over the application or theme.
--- menu.ts Contains all menu with its icons.
--- routes.ts List of components that are available in the routes module.
--- routes.module.ts Routes routing module.
-- shared Shared module amoung the entire application.
--- custom-pipes List of customized pipes.
--- colors Colors variables used in the application.
--- models List of models used in application.
--- styles Set of stylesheets used for various components and different color themes.
--- utils This module contains the list utilities for the application.
-- assets Contains the core assets which are required for the Anglar application.
-- environments It has multiple files every file points an individual environment. Each export has simple variables for the configuration.
--- environment.prod.ts Configuration for Production environment.
--- environment.ts Configuration for Development environment.
-- favicon.ico Represents the application or theme icon.
-- index.html Each and every angular application has one index.html which is the root of the application or theme. It primarily taking care of all libraries, components and modules.
-- main.ts The main entry point of any angular application or theme.
-- polyfills.ts Polyfills in angular are few lines of code which make your application compatible for different browsers. Polyfills.ts was provided by angular to help you do away with need to specifically setup everything.
-- styles.scss Global styles go here which styles that affect all of your app need to be in a central place.
-- test.ts It's a place where the unit testing starts.
.browserslistrc This file is used by the build system to adjust CSS and JS output to support the specified browsers.
.editorconfig Contains the editor configurations.
.gitignore This file used to exculde which are the files or folders not needs to be commit in the repository.
angular.json Configuration for Angular CLI. In this file you can set several defaults and also configure what files are included when your project is built. Check out the official documentation if you want to know more.
karma.conf.js The karma.conf.js file is a partial Karma configuration file. The CLI constructs the full runtime configuration in memory, based on application structure specified in the angular.json file, supplemented by karma.conf.js
package-lock.json package-lock.json is automatically generated for any operations where npm modifies either the node_modules tree, or package.json
package.json This file has meta details about the entire application. It mainly contains the libraries and its exact versions list which are used in the application.
README.md The README file used to display the general information about the theme, like Name, version and some author details.
tsconfig.app.json Visit link https://www.typescriptlang.org/docs/handbook/tsconfig-json.html for more info.
tsconfig.base.json Visit link https://www.typescriptlang.org/docs/handbook/tsconfig-json.html for more info.
tsconfig.json tsconfig.json is a file which tells the compiler about the root files structure.
tsconfig.spec.json It is TypeScript configuration for the application tests.
tslint.json Linting configuration for TSLint together with Codelyzer, used when running ng lint. Linting helps keep your code style consistent.
